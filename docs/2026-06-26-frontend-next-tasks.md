## 진행 원칙

- client 작업인지 admin 작업인지 먼저 구분한다.
- 공통 컴포넌트를 수정할 때는 양쪽 앱에 영향이 있는지 확인한다.
- 주문/회원/고객 분석 관련 작업은 백엔드 API 응답 형태와 맞춰서 진행한다.
- 이미 완료한 analytics 작업은 중복 구현하지 않고 기존 `services/analytics`와 GTM 문서를 기준으로 확장한다.
